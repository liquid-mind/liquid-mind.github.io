About

The goal of Liquid Mind is to provide a set of high-level abstractions for the Java programming language that facilitate building and maintaining complex systems.

These ideas began as a research project at the ETH Zurich and are currently under further development at the ZHAW (Zurich University of Applied Sciences in Winterthur).

Warp

The Warp persistent process engine turns normal Java threads into database backed persistent threads. These persistent processes allow for seamless integration of slow agents---typically human beings performing manual process steps---into an orchestrating process.

In the example above, authorize() is a normal Java method that may represent either a slow agent (human) or fast agent (computer) executing in an arbitrary amount of time.

Other features include a virtually unlimited number of simultaneous processes, the ability to survive JVM restarts/crashes and load balancing across multiple JVMs. To find out more or to give it a try, click on the link below.

Deflector

Turns checked exceptions into runtime exceptions by generating wrapper classes for arbitrary APIs in Java. For example: 

// JSE Class
public class InputStream {
	public abstract int read() throws IOException;
	...
}

// Deflector wrapper class
public class __InputStream {
	public abstract int read( InputStream is ) throws __IOException;
	...
}

__IOException is a runtime exception that wraps IOException, thus allowing you to invoke __InputStream.read() without the need for checked exception handling. This drastically reduces the number of exception handlers in your code.

To understand why you might want to do this, take a look at these criticisms of Java's checked exception mechanism:
